TOOL 工具層 / 04 · 爬蟲韌性

官方 API 被擋死，就改讀畫面

Dcard 的文章 API 被 Cloudflare 的人機驗證擋死，舊爬法只能無限重試、抓回 0 篇。這次換一條韌性路徑把資料救回來。

之前 · 死路

打官方 API → 被擋

→ 對每篇文章打官方 API
→ 撞上 Cloudflare 人機驗證
→ 無限重試、卡住
→ 結果：抓回 0 篇
→
之後 · 兩條韌性路徑

用固定門牌 + 直接讀畫面

1
固定 IP 出口
改用固定的對外 IP，不像旋轉 IP 那樣一連就被當機器人擋掉。
2
直接讀網頁畫面（DOM）
不靠會變的 API，改讀網頁上穩定的錨點——標題、時間、作者、版名都抓對。
為什麼重要 資料源被封鎖也能韌性續命；而且這條 proxy 路徑完全隔離，不波及 FB／PTT／Threads 其他平台。
技術細節 · 線上 e2e 通過（靜態 IP + DOM，346 秒抓回 6 篇、欄位全正確）；DOM 抽取 14/14 單元測試過；以 fetchMode 標記追蹤資料來源。
⚠ 待辦 · 靜態 IP 2026-06-26 到期（剩約 4 天），到期前需更新代理清單，否則 Dcard 會再次斷料。
