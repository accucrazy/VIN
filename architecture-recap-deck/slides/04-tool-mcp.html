把「怪名字」正名成清楚的工具契約

工具系統原本叫 MCPTool，但底下根本沒有真正的「外部工具協定」——名字和實作對不上， 導致「哪些是內建、哪些是外接」永遠講不清楚。這次把它整庫正名為 AgentTool。

之前 · Before

一個名字，兩種意思

所有工具都叫 MCPTool，但其實是內建工具。想接真正的外部工具時，邊界整個糊掉——分不清來源、也分不清給誰用。

tool: MCPTool // 名實不符
→
之後 · After

一個契約，兩個邊界

工具本體統一叫 AgentTool（什麼是工具沒變）；另外用兩個標記講清楚「從哪來、給誰用」。

source = core / plugin / mcp exposure = 給誰看得到
tool: AgentTool // 來源 × 用途 正交
為什麼重要 消除「內建 vs 外接」的邊界混淆，替日後真正接上外部工具鋪好乾淨的地基——而且不動到任何現有執行路徑。
